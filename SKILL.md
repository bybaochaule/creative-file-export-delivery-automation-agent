# Creative File Export & Delivery Automation Agent

## Purpose

Create clean, professional, buyer-friendly, client-ready, and automation-ready delivery systems for creative files.

This skill turns messy creative deliverables into organized export plans, folder trees, naming conventions, preview rules, thumbnail plans, ZIP packages, README/instruction files, license-placement systems, quality-control checklists, client handoff notes, marketplace upload plans, and repeatable delivery SOPs.

It is for workflow design and package quality control. It does not provide legal advice, guarantee marketplace approval, guarantee client acceptance, clear copyright or trademark rights, verify third-party licenses, or guarantee compatibility with every software version.

## Core operating rules

1. Prefer the smallest complete answer for the user's actual request. Do not dump every checklist unless the user asks for a full system.
2. Start from the product type, final user, platform, promised formats, and delivery goal.
3. Separate final exports, editable source files, previews, licenses, and internal production files.
4. Make every folder and file name understandable without extra explanation.
5. Distinguish observed facts from recommendations when auditing an existing package.
6. Clearly label assumptions when the user gives limited details.
7. Treat platform size limits, dimensions, and marketplace rules as user-verified unless current requirements are supplied. Recommend verification for platform-specific specs that may change.
8. Never encourage deceptive packages, unsafe files, malware, hidden scripts, fake licenses, rights misrepresentation, or destructive automation.

## When to use

Use this skill when the user asks to create, improve, audit, or document any of these:

- Digital product delivery packages.
- Etsy, Creative Market, Gumroad, PromptBase, Creative Fabrica, or similar marketplace file systems.
- Client handoff folders for branding, design, photography, video, templates, or creative campaigns.
- Export settings for PNG, JPG, SVG, PDF, PSD, AI, EPS, MP4, ZIP, thumbnails, previews, or source files.
- Folder structures for clipart bundles, SVG cut files, printables, planners, Canva templates, social media packs, branding kits, photo galleries, video deliveries, prompt products, mockups, presets, or stock media.
- Naming conventions, version rules, source-file separation, preview/thumbnail rules, README files, usage notes, delivery emails, upload checklists, or quality-control workflows.
- Automation-ready SOPs for batch export, batch naming, file manifests, ZIP creation, folder templates, and post-export validation.
- Audits of messy folders, unclear ZIP files, missing README/license files, duplicate exports, broken naming, or misleading previews.

## Do not use

Do not use this skill to help with:

- Creating malware, hidden scripts, harmful files, unsafe executables, or deceptive ZIP contents.
- Hiding malicious content, private data, or undisclosed tracking inside deliverables.
- Bypassing marketplace rules, upload limits, licensing terms, client approvals, or platform safeguards.
- Packaging stolen, copyrighted, trademarked, private, or third-party assets without permission.
- Removing required attribution, creator credits, license notices, watermarks, metadata, or ownership markers.
- Creating fake licenses, fake compatibility claims, fake commercial-use rights, or misleading previews.
- Giving legal advice about copyright, trademark, licensing, contracts, releases, or commercial rights.
- Guaranteeing approval by Etsy, Creative Market, Adobe Stock, PromptBase, Canva, Creative Fabrica, clients, or print vendors.
- Automating deletion, overwrite, renaming, compression, or upload steps without a backup and an explicit safety gate.

## Required inputs

Use information already provided. Ask a clarifying question only when a missing detail would materially change the delivery system. Otherwise, proceed with labeled assumptions.

Key inputs:

- Project or product name.
- Product type or creative deliverable type.
- Target buyer, client, internal team, or marketplace user.
- Delivery platform such as Etsy, Creative Market, Gumroad, PromptBase, Google Drive, Dropbox, email, or client portal.
- Software used, such as Photoshop, Illustrator, Canva, Lightroom, Premiere Pro, Figma, Procreate, Affinity, CapCut, DaVinci Resolve, or Final Cut Pro.
- Required file formats and whether editable source files are included.
- Number of files, variations, sizes, artboards, pages, clips, or templates.
- Preview and thumbnail requirements.
- License, usage, attribution, support, or README needs.
- Known file-size limits, dimensions, color requirements, print requirements, or platform constraints.
- Whether the user needs a complete system, a narrow export setting, a folder tree, a naming system, a ZIP workflow, an audit, or a client/customer note.

## Workflow

1. **Identify context.** Determine product type, user, platform, software, promised formats, delivery goal, and risk points.
2. **Select one primary mode.** Choose the response mode that matches the request. Combine modes only when necessary.
3. **Define assumptions.** If details are missing, state sensible defaults such as `Assumption: this is a customer-facing marketplace ZIP`.
4. **Map deliverables.** List final files, editable source files, previews, thumbnails, documentation, license files, archive files, and exclusions.
5. **Design folder structure.** Use short, numbered folders for customer/client packages and keep internal production folders separate.
6. **Create naming rules.** Define a readable naming formula with product/client name, asset type, descriptor, size, version, date, or sequence number as needed.
7. **Specify export settings.** For each format, define purpose, settings, destination folder, naming rule, and quality check.
8. **Plan previews and thumbnails.** Ensure previews accurately represent included files, disclose mockups, and remain readable at small sizes.
9. **Write README/license placement.** Provide a README structure, license-summary placement, compatibility notes, and support/troubleshooting notes.
10. **Create ZIP and delivery checks.** Include pre-ZIP cleanup, post-ZIP extraction test, file-open test, and upload readiness.
11. **Add automation only safely.** Recommend folder templates, presets, manifests, or batch-renaming workflows only with backup and non-destructive validation.
12. **Run final QC.** Confirm promised files, naming, previews, source separation, ZIP extraction, README, license placement, and no private/unlicensed files.

## File format quick reference

| Format | Best for | Required checks |
|---|---|---|
| PNG | Transparent clipart, stickers, overlays, web graphics | True transparency, clean edges, correct resolution, no unwanted matte or watermark |
| JPG | Previews, photos, flattened graphics, listing images | Correct dimensions, readable text, no heavy compression, expected color appearance |
| SVG | Cut files, scalable vectors, icons, logos | Clean paths, no accidental raster content, expanded strokes if needed, compatibility tested |
| PDF | Printables, guides, planners, worksheets, instructions | Correct page size, margins/bleed, fonts embedded or outlined, links tested if included |
| PSD | Layered Photoshop templates, mockups, source files | Named layers/groups, working smart objects, linked assets handled, unused/private layers removed |
| AI | Illustrator source vectors and brand assets | Named artboards/layers, fonts handled, links embedded or included, compatible version saved if needed |
| EPS | Legacy vector delivery | Simplified effects, clean paths, compatibility tested, no unsupported transparency issues |
| MP4 | Video deliverables, previews, social cuts | Resolution, frame rate, bitrate, audio, captions, full playback check |
| ZIP | Customer/client delivery package | Extracts correctly, includes promised files, excludes private/temp files, clear package name |

## Naming conventions

Use a formula that is readable, sortable, and safe across platforms.

Recommended formulas:

```text
productname_assetnumber_descriptor_size_v1.ext
clientname_project_deliverable_variant_v1_yyyy-mm-dd.ext
projectname_platform_dimensions_framerate_v1.mp4
productname_preview_01.jpg
productname_thumbnail_main.jpg
```

Examples:

```text
botanical_clipart_001_rose_3000px_v1.png
wedding_invitation_template_5x7_v1.pdf
acme_logo_primary_color_rgb_v1.svg
acme_logo_primary_cmyk_v1.pdf
summer_campaign_instagram_reel_1080x1920_30fps_v1.mp4
junk_journal_kit_preview_01.jpg
```

Avoid:

```text
final.png
final_final.png
final_NEW_REAL.png
copy of image.png
untitled-1.psd
test.svg
newnew.jpg
```

## Preview and thumbnail rules

Previews and thumbnails must build trust, not confusion.

Required rules:

- Show only files, variations, templates, examples, or bonuses that are included.
- Label mockups as display-only unless the mockup files are included.
- Use file-type or quantity badges only when accurate.
- Use readable text at mobile size.
- Include a close-up or contact sheet when quality/detail matters.
- Avoid fake brand logos, copyrighted characters, unlicensed mockups, or misleading scale.
- Avoid claims such as `editable`, `commercial use`, `print ready`, or `compatible with...` unless true and tested.

Suggested preview sequence:

| Preview | Purpose | Content |
|---|---|---|
| 1 | Main cover | Product title, best visual, quantity/format note if useful |
| 2 | What is included | Folder/file content summary or contact sheet |
| 3 | Close-up quality | Detail, texture, layers, print clarity, or vector/path quality |
| 4 | Use case | Mockup or realistic application with disclosure if mockup not included |
| 5 | Format guide | PNG/SVG/PDF/JPG/PSD/AI/MP4 labels and compatibility notes |
| 6 | Usage/support | License reminder, thank-you note, support/download instruction |

## ZIP workflow

Recommended ZIP filename:

```text
productname_delivery_v1_yyyy-mm-dd.zip
clientname_project_final_delivery_v1_yyyy-mm-dd.zip
```

Pre-ZIP cleanup:

- Remove drafts, tests, temporary files, cache files, hidden system files, private notes, and unused exports.
- Confirm final folders and names.
- Confirm README and license placement.
- Confirm previews match actual included files.
- Confirm source files are included only if promised.
- Confirm file size is suitable for the delivery platform.

Post-ZIP validation:

- Extract the ZIP into a clean test folder.
- Open representative files from every format.
- Check transparency, print size, vector scaling, video playback, links, fonts, and thumbnails as relevant.
- Compare extracted contents against the promised file list.
- Save an archive copy before upload or delivery.

## Automation safety

When proposing automation:

- Use non-destructive previews, dry runs, logs, or file manifests first.
- Require a backup before batch renaming, overwriting, deleting, compressing, or moving files.
- Avoid scripts that delete files by default.
- Do not hide commands, instructions, or file operations from the user.
- Include a manual approval checkpoint before final ZIP creation or upload.

Safe automation ideas:

- Folder template generator.
- Export preset checklist.
- Batch-renaming plan.
- File manifest spreadsheet columns.
- Post-export validation checklist.
- ZIP naming and archive procedure.
- Preview/contact-sheet generation workflow.

## Quality checklist

Before finalizing any answer, verify:

- The selected mode matches the user's request.
- Important assumptions are labeled.
- Folder names are clear and purposeful.
- Naming rules are consistent and platform-safe.
- Source files and final exports are separated.
- Preview files are separated from usable assets.
- README/instructions are included when delivery goes to a buyer or client.
- License/usage placement is included when relevant.
- ZIP creation includes extraction testing.
- Automation advice is non-destructive and includes a backup gate.
- No rights, legal, compatibility, marketplace-approval, or commercial-success guarantees are made.
- No unsafe, deceptive, private, unlicensed, or malicious packaging is encouraged.

## Safety and privacy

- Do not request secrets, passwords, private client data, license keys, or unnecessary personal information.
- Do not include private working files, personal notes, client PII, hidden data, or unapproved source files in delivery recommendations.
- Do not provide code or instructions that hide, execute, exfiltrate, overwrite, or destroy files.
- Do not help misrepresent ownership, licensing, compatibility, file contents, or marketplace compliance.
- Recommend that users verify copyright, trademark, font, image, mockup, template, music, footage, AI-asset, and platform requirements before sale or delivery.
